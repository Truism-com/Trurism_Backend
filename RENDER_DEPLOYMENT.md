# Render Deployment Guide for FastAPI Travel Booking Platform

This guide explains what has been configured for Render deployment and what you need to do to deploy your FastAPI application.

## ✅ What's Already Configured

### 1. **Dockerfile** ✓
- Multi-stage build with all necessary system dependencies
- Installs Python packages from `requirements.txt`
- Includes startup script that runs database migrations automatically
- Properly handles `PORT` environment variable from Render

### 2. **render.yaml** ✓
- Web service configuration for FastAPI API
- Worker service configuration for Celery (optional)
- PostgreSQL database configuration
- Environment variables setup with database linking
- Automatic database URL conversion from `postgresql://` to `postgresql+asyncpg://`

### 3. **Configuration Updates** ✓
- **Database URL Conversion**: Automatically converts Render's `postgresql://` to `postgresql+asyncpg://` for asyncpg compatibility
- **CORS Configuration**: Made configurable via `CORS_ORIGINS` environment variable
- **Trusted Hosts**: Made configurable via `TRUSTED_HOSTS` environment variable
- **Automatic Migrations**: Database migrations run automatically on startup via `start.sh`

### 4. **Startup Script** ✓
- `start.sh` runs Alembic migrations before starting the application
- Handles errors gracefully if migrations fail
- Starts Uvicorn with proper host and port configuration

## 🚀 Deployment Steps

### Step 1: Push to GitHub/GitLab

Ensure your code is pushed to a Git repository:

```bash
git add .
git commit -m "Prepare for Render deployment"
git push origin main
```

### Step 2: Create Render Account and Connect Repository

1. Sign up/Login at [render.com](https://render.com)
2. Click **"New +"** → **"Blueprint"** (if using render.yaml) or **"Web Service"**
3. Connect your GitHub/GitLab repository
4. If using Blueprint:
   - Select **"Use a render.yaml"** option
   - Render will read `render.yaml` and create all services automatically
5. If deploying manually:
   - Select the repository
   - Choose **"Docker"** as the environment
   - Point to your `Dockerfile`

### Step 3: Configure Environment Variables

In Render Dashboard → Your Web Service → Environment:

#### Required Variables:

- **`DATABASE_URL`**: Automatically set if you use `fromDatabase` in render.yaml
- **`JWT_SECRET_KEY`**: Auto-generated by Render (or set manually with a strong secret)
- **`DEBUG`**: Set to `false` for production
- **`ENVIRONMENT`**: Set to `production`

#### Optional Variables:

- **`REDIS_URL`**: Create a Redis instance in Render Dashboard and copy the connection URL
  - Steps:
    1. Go to Render Dashboard
    2. Click **"New +"** → **"Redis"**
    3. Choose a name and plan
    4. Copy the **Internal Redis URL** or **External Redis URL**
    5. Set it in your web service environment variables as `REDIS_URL`

- **`CELERY_BROKER_URL`**: Usually same as `REDIS_URL` with database number (e.g., `redis://...6379/1`)
- **`CELERY_RESULT_BACKEND`**: Usually same as `REDIS_URL` with database number (e.g., `redis://...6379/2`)

- **`CORS_ORIGINS`**: Comma-separated list of allowed origins (e.g., `https://yourdomain.com,https://www.yourdomain.com`)
  - If not set, defaults to `["*"]` (allows all origins)
  - For production, specify your frontend domains

- **`TRUSTED_HOSTS`**: Comma-separated list of trusted hosts (e.g., `yourdomain.com,*.yourdomain.com`)
  - If not set, defaults to `["*"]` (allows all hosts)
  - For production, specify your domain

#### External API Variables (if needed):

- `XML_AGENCY_USERNAME`: Your XML Agency API username
- `XML_AGENCY_PASSWORD`: Your XML Agency API password
- `RAZORPAY_KEY_ID`: Razorpay key ID (if using payment gateway)
- `RAZORPAY_KEY_SECRET`: Razorpay key secret (if using payment gateway)

### Step 4: Database Setup

If using `render.yaml` with Blueprint:
- Database is automatically created and linked
- Connection string is automatically set in `DATABASE_URL`
- Database URL is automatically converted to asyncpg format

If deploying manually:
1. Create a PostgreSQL database in Render Dashboard
2. Go to **"New +"** → **"PostgreSQL"**
3. Choose a name and plan (Starter plan is fine for testing)
4. Copy the **Internal Database URL** or **External Database URL**
5. Convert it: Change `postgresql://` to `postgresql+asyncpg://`
6. Set it as `DATABASE_URL` in your web service environment variables

### Step 5: Deploy

1. Click **"Create Web Service"** or **"Apply"** (if using Blueprint)
2. Render will:
   - Build the Docker image
   - Run the startup script (which runs migrations)
   - Start your FastAPI application
3. Wait for deployment to complete
4. Your API will be available at `https://your-service-name.onrender.com`

### Step 6: Verify Deployment

1. **Health Check**: Visit `https://your-service-name.onrender.com/health`
   - Should return `{"status": "healthy", ...}`

2. **Root Endpoint**: Visit `https://your-service-name.onrender.com/`
   - Should return API information

3. **API Documentation** (if DEBUG=true):
   - Visit `https://your-service-name.onrender.com/docs`
   - Note: Docs are disabled in production (DEBUG=false)

## 🔧 Key Features for Render

### Automatic Database URL Conversion

The application automatically converts Render's PostgreSQL connection string from:
```
postgresql://user:pass@host:5432/dbname
```
to:
```
postgresql+asyncpg://user:pass@host:5432/dbname
```

This is handled in `app/core/config.py` via the `post_init_processing` method.

### Automatic Migrations

On startup, the `start.sh` script runs:
```bash
alembic upgrade head
```

This ensures your database schema is always up-to-date when the service starts.

### Port Configuration

The application automatically uses Render's `PORT` environment variable:
- Render provides `$PORT` at runtime
- The startup script uses `uvicorn ... --port ${PORT:-8000}`
- Falls back to port 8000 if `$PORT` is not set (shouldn't happen on Render)

## 🐛 Troubleshooting

### Database Connection Issues

**Problem**: "Database connection failed"

**Solutions**:
- Check that `DATABASE_URL` is set correctly
- Verify database URL format: should be `postgresql+asyncpg://...`
- Check database is running in Render Dashboard
- Verify database credentials are correct

### Migration Failures

**Problem**: Migrations fail on startup

**Solutions**:
- Check deployment logs for specific error
- Ensure all migration files are committed to repository
- Try running migrations manually in Render Shell:
  1. Go to your service → "Shell"
  2. Run: `alembic upgrade head`

### Redis Connection Issues

**Problem**: Redis health check fails

**Solutions**:
- Verify `REDIS_URL` is set correctly
- Check Redis instance is running in Render Dashboard
- Redis is optional - app will continue without it (caching won't work)

### CORS Issues

**Problem**: Frontend can't access API

**Solutions**:
- Set `CORS_ORIGINS` environment variable with your frontend domain
- Ensure frontend domain matches exactly (including https/http)
- Check CORS middleware is configured correctly in logs

## 🔐 Security Best Practices

1. **Never commit `.env` files** - Use Render's environment variables
2. **Use strong JWT_SECRET_KEY** - Let Render generate it or use a secure random string
3. **Set DEBUG=false** in production
4. **Configure CORS_ORIGINS** - Don't use `["*"]` in production
5. **Configure TRUSTED_HOSTS** - Restrict allowed hosts in production
6. **Use SSL/TLS** - Render provides this automatically
7. **Keep dependencies updated** - Regularly update `requirements.txt`

## 📝 Notes

- **Free Tier Limitations**: Render's free tier has limitations (spins down after inactivity, limited resources)
- **Database**: Starter plan PostgreSQL is suitable for development/testing
- **Redis**: Not included in render.yaml (must be created manually or use external Redis)
- **Worker Service**: Celery worker service is included but disabled by default (uncomment in render.yaml if needed)
- **Auto-Deploy**: Render auto-deploys on git push to main branch (configurable)
